# CS300

What was the problem you were solving in the projects for this course?

This course introduced us to numerous types of data structures and ways to manipulate them. The problem we were asked to solved was which data structure was most efficient at storing, sorting, and printing a list of data we were given based on run-times and ease of manipulating the structure. 

How did you approach the problem? Consider why data structures are important to understand.

We were given the opportunity to practice with each type of data structure during our assignments. I took that time and also my run-time analysis into account in order to decide which structure I thought would be most appropriate. It is important to understand how data structures work because the way that they perform certain activities, like search a list, greatly effect their run-time and how effcient they are to use. 

How did you overcome any roadblocks you encountered while going through the activities or project?

Anytime I ran into a roadblock with my coding I referred back to the pseudocode that I had written and compared that to my code to stay on track or fix what I had missed. 

How has your work on this project expanded your approach to designing software and developing programs?

This project was the first where I learned about run-times and how the efficiency of a data structure really affected how the entire program ran, and how that changes when the data structure is provided with significantly larger amounts of data. In the future I will be able to implement the correct data structure for the program's needs because of everything I learned during this project. 

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

During this project I learned not only how to choose the most efficient data structure but also got to practice implementing all that I have learned about using good coding practices so far. For example, ensuring readability by using informative succinct comments and organized code, built distinct functions that someone could call without having to modify the code, and obviously ensured that the code functioned correctly.
